価格・表示ルール

合計・レビュー計算に使う価格の扱い。

月額プラン

金額と月あたりコストは同額。ホーム集計にはそのまま反映する。

月あたり = 月額

年額プラン

年額金額と月あたりコストを併記。ホームでは比較しやすい月あたりを優先表示。

月あたり = 年額 / 12

週額プラン

週額のままだと比較しづらいため、月あたりに換算してレビューへ反映する。

月あたり = 週額 × 52 / 12

任意周期

何日ごと、何か月ごとなどの間隔を保存し、周期入力後に月あたりコストを計算する。

カタログ価格

登録時の初期値として使う。カタログ更新後も、保存済み価格は自動変更しない。

手修正価格

ユーザーが修正した金額を effectivePrice として扱い、合計・レビュー計算に使う。

実装前タスク

Android実装に入る前に決めることと、環境構築の最短手順。

MVP範囲

  • 初期実装は S01 / S02 / S03 / S05 / S07 / S08 に絞る。
  • S03でプラン選択、金額編集、設定完了まで行う。
  • 銀行連携、Gmail解析、共有機能は後回しにする。

データ設計

  • 既知サービスカタログとユーザー登録データを分ける。
  • カタログ価格は初期値、手修正価格を実支出として扱う。
  • 年額・週額・任意周期は月あたりコストを計算する。

Android技術スタック

  • Kotlin + Jetpack Compose。
  • Navigation Composeで画面遷移を管理。
  • Roomでローカル保存。
  • 通知はWorkManagerまたはAlarmManagerで検討。

環境構築

  • Android Studio安定版をインストール。
  • 初回起動でAndroid SDK / Platform Toolsを導入。
  • 導入後、ターミナルを開き直してadbを確認。
C:\LLM\codex\sample\setup-android-env.cmd

Codex CLIで自動化する作業

  • Composeプロジェクト作成。
  • 画面雛形とNavigation構成の作成。
  • Room Entity / DAO / Repositoryの作成。
  • 初期サービスカタログJSONの作成。
codex "app-design.html に沿ってAndroid Composeプロジェクトの雛形を作成してください"
